// ThreadSafeFunction + Promise pattern for async bridge calls
Napi::Value GetEngineInfo(const Napi::CallbackInfo& info) {
    Napi::Env env = info.Env();
    auto deferred = Napi::Promise::Deferred::New(env);
    auto tsfn = Napi::ThreadSafeFunction::New(env, ...);
    std::thread([deferred, tsfn]() {
        // call C++ engine
        tsfn.BlockingCall([deferred, result](Napi::Env env, Napi::Function) {
            deferred.Resolve(/* napi value */);
        });
        tsfn.Release();
    }).detach();
    return deferred.Promise();
}

**2. Implement bridge functions in `native/src/bridge.cpp`:**

Add `#include "calliope/engine.h"` at top (already included).

Follow the existing ThreadSafeFunction + Promise pattern for each function. The key difference from Phase 1: most Phase 2 operations are fast (atomic reads/writes) and don't need a separate thread. However, `InitialiseEngine` and `ShutdownEngine` DO need the thread because they touch AudioDeviceManager (blocking I/O). For consistency and safety, use the thread pattern for ALL functions.
