```cpp
blockBuffer.setSize(2, samplesThisBlock, false, false, true);

blockBuffer.setSize(2, samplesThisBlock, false, false, true);  // line 177

graph.getMasterBus().processBlock(blockBuffer, midiBuffer);  // line 181 — second call on CLEARED buffer

// Copy from second (cleared) call — always silence

                   blockBuffer, ch, 0, samplesThisBlock);

```

Replace with:

```cpp
blockBuffer.setSize(2, samplesThisBlock, false, false, true);
blockBuffer.clear();
graph.getMasterBus().processBlock(blockBuffer, midiBuffer);

// Copy processed samples to output
for (int ch = 0; ch < 2; ++ch) {
    output.copyFrom(ch, static_cast<int>(samplesRendered),
                   blockBuffer, ch, 0, samplesThisBlock);
}
```

Key changes:
1. Clear the buffer BEFORE the single processBlock call (processBlock writes into the buffer)
2. Remove the second processBlock call entirely
3. Remove the redundant setSize between the two old calls
4. Remove the misleading comment about "actually use the full graph"

Do NOT change anything else in offlineBounce — the MIDI scheduling, transport advancing, progress callback, and surrounding code are all correct.
