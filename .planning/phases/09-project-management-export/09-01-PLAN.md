<objective>
Implement project save/load and autosave for the DAW. The C++ engine serializes full project state (transport, instruments, effects, timeline data) to a versioned JSON file. The Electron main process handles file dialogs and autosave timer. The renderer tracks project dirty state.

Purpose: After this plan, users can save their work to disk, reload it later, and rely on autosave for crash protection.
Output: ProjectSerializer C++ class, bridge/IPC/preload wiring, Zustand project store, autosave timer in main process.
</objective>

<task id="1" type="auto">
<name>C++ ProjectSerializer and bridge wiring for save/load</name>
<description>
Create a ProjectSerializer class that writes ProjectState to disk as versioned JSON and reads it back. Add bridge functions for saveProject(filePath) and loadProject(filePath). Wire IPC handlers in main process. Include Electron file dialog support (showSaveDialog / showOpenDialog).

Implementation details:
1. engine/include/calliope/project_serializer.h - Static methods: saveToFile(filePath, projectState), loadFromFile(filePath) -> ProjectState
2. engine/src/project_serializer.cpp - Wraps ProjectState::toJson/fromJson with file I/O and version envelope {"version": 1, "appVersion": "1.0.0", "data": {...}}
3. native/src/bridge.cpp - Add SaveProject and LoadProject bridge functions that get ProjectState from Engine, serialize, and write. For load: read file, deserialize, apply to Engine.
4. app/src/main/native-bridge.ts - Add saveProject(filePath: string) and loadProject(filePath: string) to NativeAddon interface
5. app/src/main/index.ts - Add IPC handlers for project:save, project:saveAs, project:load with Electron dialog integration
6. app/src/preload/index.ts - Expose projectSave, projectSaveAs, projectLoad, projectNew methods
7. app/src/renderer/types/calliope.d.ts - Update CalliopeAPI type with project methods
</description>
<done_criteria>Bridge builds. IPC handlers registered. Preload exposes project save/load/new methods.</done_criteria>
</task>

<task id="2" type="auto">
<name>Autosave timer, project store, and dirty tracking</name>
<description>
Create a Zustand project store that tracks project metadata (filePath, isDirty, lastSaved, autosave settings). Implement autosave in the Electron main process using setInterval. Wire dirty tracking to command events.

Implementation details:
1. app/src/renderer/stores/project-store.ts - Zustand store with: filePath, isDirty, lastSaved, autosaveEnabled (default true), autosaveIntervalMs (default 120000), projectName. Actions: markDirty, markClean, setFilePath, setAutosave, reset (new project).
2. app/src/main/index.ts - Add autosave timer logic: when autosave enabled and project has a filePath and isDirty, trigger save. Use IPC to communicate autosave state. Add project:setAutosave and project:getAutosaveConfig IPC handlers.
3. app/src/preload/index.ts - Expose setAutosave(enabled, intervalMs) and getAutosaveConfig()
4. Wire command events to mark project dirty - when command:event fires, project becomes dirty
5. After save completes, mark project clean and update lastSaved
</description>
