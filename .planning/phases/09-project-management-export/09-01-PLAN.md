<objective>
Implement project save/load with full state restoration and autosave.

Purpose: Users need to persist their work across sessions. Save captures both C++ engine state (instruments, effects, transport) and Zustand UI state (tracks, clips, MIDI notes, mixer volumes/pans) into a single .calliope JSON file. Load restores everything. Autosave prevents data loss.

Output: Complete save/load/autosave pipeline from renderer through IPC to disk, with dirty tracking and unsaved changes protection.
</objective>

    - `saveProject(filePath: string, uiState: object): Promise<void>` — Gets engine state via `native.getProjectState()`, merges into `{ version: 1, appVersion: app.getVersion(), savedAt: new Date().toISOString(), engine: JSON.parse(engineJson), ui: uiState }`, writes with `fs.promises.writeFile` (async, never sync).     - Expose `window.__getProjectUIState` function that collects state from all Zustand stores (timeline, mixer, piano-roll) and returns it. Handle Set -> Array conversion for selectedClipIds/selectedNoteIds. Skip peakData (Float32Array) — regenerate on load per Research Pitfall 6.
