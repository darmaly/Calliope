**ClipScheduler C++ class dispatches MIDI events from clips to instruments during audio callback, with full IPC pipeline from renderer through preload/bridge to engine**

## Accomplishments
- ClipScheduler C++ class with CRUD operations and audio-thread processBlock that dispatches MIDI noteOn/noteOff to instruments based on transport position
- Full IPC pipeline: renderer clip-operations -> preload clipAdd/clipRemove/clipUpdate/clipClear -> IPC handlers -> bridge AddClip/RemoveClip/UpdateClip/ClearClips -> ClipScheduler
- 6 Catch2 unit tests (10 assertions) all passing for clip scheduler
- TypeScript compiles cleanly with electron-vite build

## Decisions Made
- Used mutex-protected std::map for clip storage (safe for bridge-thread writes, audio-thread reads via processBlock)
- Before/after ID diff pattern to capture Zustand-generated clip IDs without modifying store interface
- Direct preload API (clipAdd etc.) rather than routing through command dispatcher -- clip operations are not undoable via CommandDispatcher
