    **Effect commands** (engine/include/calliope/commands/effect_commands.h and .cpp):

    All commands take `InsertChain&` and `Engine&` references. The Engine reference is needed for parameter registration/unregistration.

    1. **InsertEffectCommand**:
       - Constructor: `InsertEffectCommand(InsertChain& chain, Engine& engine, const juce::String& trackId, const juce::String& effectType, int position)`
       - `effectType` is one of: "eq", "compressor", "reverb", "delay", "limiter"
       - `perform()`: Create the appropriate AudioProcessor based on effectType using a factory function `createEffect(effectType)`. Call `chain_.insertEffect(std::move(effect), position_)`. Store the actual insertion index. Call `engine_.registerEffectParameters(trackId_, insertedIndex_, chain_.getEffect(insertedIndex_))`.
       - `undo()`: Call `engine_.unregisterEffectParameters(trackId_, insertedIndex_, ...)`. Call `chain_.removeEffect(insertedIndex_)`. Store the removed processor in `removedEffect_` for potential redo.
       - `isUndoable()` returns true
       - `getEventData()` returns JSON with effectType, trackId, position
       - Factory function `static std::unique_ptr<juce::AudioProcessor> createEffect(const juce::String& type)` creates the right processor

    2. **RemoveEffectCommand**:
       - Constructor: `RemoveEffectCommand(InsertChain& chain, Engine& engine, const juce::String& trackId, int position)`
       - `perform()`: Store effect type name before removal. Unregister parameters. Call `chain_.removeEffect(position_)`, store result in `removedEffect_`.
       - `undo()`: Re-insert `removedEffect_` at `position_`. Re-register parameters.
       - `isUndoable()` returns true

    3. **ReorderEffectCommand**:
       - Constructor: `ReorderEffectCommand(InsertChain& chain, Engine& engine, const juce::String& trackId, int fromPosition, int toPosition)`
       - `perform()`: Call `chain_.moveEffect(fromPosition_, toPosition_)`. Unregister and re-register ALL effect parameters for this trackId (since slot indices shift).
       - `undo()`: Call `chain_.moveEffect(toPosition_, fromPosition_)`. Re-register all parameters.
       - `isUndoable()` returns true

    4. **BypassEffectCommand**:
       - Constructor: `BypassEffectCommand(InsertChain& chain, int position, bool bypassed)`
       - `perform()`: Store previous bypass state. Call `chain_.setBypass(position_, bypassed_)`.
       - `undo()`: Restore previous bypass state.
       - `isUndoable()` returns true
