    Create the effects/ subdirectories under engine/include/calliope/ and engine/src/.

    **All 5 effect processors follow the same AudioProcessor subclass pattern as MasterBusProcessor:**

    1. **ParametricEqProcessor** (FX-01):
       - 4 bands: band[0]=lowShelf (100Hz), band[1]=peak (500Hz), band[2]=peak (2000Hz), band[3]=highShelf (8000Hz)
       - Each band has: `std::atomic<float> frequency`, `std::atomic<float> q` (default 0.71), `std::atomic<float> gainDb` (default 0.0), `std::atomic<bool> enabled` (default true)
       - Use a struct `BandParams` for each band, store as `BandParams bands[4]`
       - Internal: `juce::dsp::IIR::Filter<float> filters_[4][2]` (4 bands x 2 channels)
       - In `prepareToPlay`: reset all filters, store sampleRate
       - In `processBlock`: if bypassed return early. Call `updateFilters()` to recalc coefficients, then process each enabled band's filters per channel using `juce::dsp::AudioBlock` and `ProcessContextReplacing`
       - `updateFilters()`: use `dsp::IIR::ArrayCoefficients<float>::makeLowShelf` for band 0, `makePeakFilter` for bands 1-2, `makeHighShelf` for band 3. Apply via `*filters_[band][ch].coefficients = *newCoeffs`
       - Global `std::atomic<bool> bypassed{false}`
       - `getTailLengthSeconds()` returns 0.0

    2. **CompressorProcessor** (FX-02):
       - Atomic params: `threshold{-10.0f}` (dB), `ratio{4.0f}`, `attack{10.0f}` (ms), `release{100.0f}` (ms), `makeupGain{0.0f}` (dB), `bypassed{false}`
       - Internal: `juce::dsp::Compressor<float> compressor_`
       - In `processBlock`: if bypassed return. Set compressor params from atomics, process via `dsp::ProcessContextReplacing`, then apply makeup gain via `buffer.applyGain(Decibels::decibelsToGain(makeupDb))` if non-zero
       - `getTailLengthSeconds()` returns 0.0

    3. **ReverbProcessor** (FX-03):
       - Atomic params: `roomSize{0.5f}`, `damping{0.5f}`, `wetLevel{0.33f}`, `dryLevel{0.67f}`, `preDelay{0.0f}` (ms), `bypassed{false}`
       - Internal: `juce::dsp::Reverb reverb_`, `juce::dsp::DelayLine<float, juce::dsp::DelayLineInterpolationTypes::Linear> preDelayLine_{48000}` (1 second max pre-delay)
       - In `processBlock`: if bypassed return. Copy atomic params to `juce::Reverb::Parameters` struct, call `reverb_.setParameters()` then `reverb_.process()`. If preDelay > 0, process through preDelayLine_ first using pushSample/popSample per sample per channel.
       - `getTailLengthSeconds()` returns 2.0 (reverb tail)

    4. **DelayProcessor** (FX-04):
       - Atomic params: `feedback{0.3f}`, `wetDry{0.5f}`, `syncNoteValue{1.0f}` (1.0=quarter, 0.5=eighth, 0.25=sixteenth, 1.5=dotted quarter, 0.667=triplet quarter), `pingPongEnabled{false}`, `bypassed{false}`
       - `std::atomic<float> currentBpm_{120.0f}` and method `void setCurrentBpm(float bpm)`
       - Internal: `juce::dsp::DelayLine<float, juce::dsp::DelayLineInterpolationTypes::Linear> delayLines_[2]` (one per channel, max 576000 samples for 12 seconds at 48kHz), `juce::dsp::DryWetMixer<float> dryWetMixer_`
       - Feedback MUST be clamped to [0.0, 0.98] in processBlock
       - `getDelayTimeInSamples()`: `(syncNoteValue / (bpm / 60.0)) * sampleRate`
       - In `processBlock`: if bypassed return. Push dry samples to mixer, process per-sample with delay lines and feedback. For ping-pong: cross-feed L feedback to R delay line and vice versa. Then mixWetSamples.
       - `getTailLengthSeconds()` returns 12.0 (max delay time)

    5. **LimiterProcessor** (FX-05):
       - Atomic params: `threshold{-1.0f}` (dB), `release{100.0f}` (ms), `bypassed{false}`
       - Internal: `juce::dsp::Limiter<float> limiter_`
       - In `processBlock`: if bypassed return. Set limiter params from atomics, process via `dsp::ProcessContextReplacing`
       - `getTailLengthSeconds()` returns 0.0

    **All effects share these AudioProcessor stubs** (copy from MasterBusProcessor):
    - `acceptsMidi()` returns false, `producesMidi()` returns false
    - `hasEditor()` returns false, `createEditor()` returns nullptr
    - `getNumPrograms()` returns 1, `getCurrentProgram()` returns 0
    - `setCurrentProgram()`, `changeProgramName()`, `getProgramName()`, `getStateInformation()`, `setStateInformation()` are no-ops

    **InsertChain class:**
    - NOT an AudioProcessor subclass. Standalone class used inside InsertChainProcessor (plan 02).
    - Uses double-buffer pattern for real-time safety: `ChainState` struct containing `std::vector<std::unique_ptr<juce::AudioProcessor>>` and `std::vector<bool> bypassed`
    - `activeChain_` is `std::atomic<ChainState*>` read by audio thread
    - `chainA_` and `chainB_` are `std::unique_ptr<ChainState>` owned by message thread
    - `processBlock()`: load activeChain_ with acquire, iterate effects, skip bypassed, call each effect's processBlock
    - `prepareToPlay(double sampleRate, int samplesPerBlock)`: store sampleRate/blockSize, prepare all effects in active chain
    - `insertEffect(std::unique_ptr<AudioProcessor>, int position)`: build new chain in inactive buffer, prepare the new effect, atomically swap. Returns the slot index.
    - `removeEffect(int position)`: returns `std::unique_ptr<AudioProcessor>` (the removed effect). Build new chain without it, swap.
    - `moveEffect(int from, int to)`: reorder in inactive buffer, swap
    - `setBypass(int position, bool)`: update bypass state, swap
    - `isBypassed(int position) const`, `getNumEffects() const`, `getEffect(int position) const` (returns raw pointer)
