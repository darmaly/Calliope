## Visual Hierarchy

**Primary focal point:** Transport bar is always visible and provides the primary temporal context (where am I, what is playing, at what tempo).

**Reading order:**
1. **Transport bar** (top, fixed) -- play state, position, tempo
2. **Timeline** (main area) -- arrangement context, clip layout
3. **Bottom panels** (piano roll / mixer) -- detail editing when open

**Panel focus system:**
- Exactly one panel is "focused" at a time (timeline, piano roll, or mixer)
- Click anywhere within a panel to focus it
- Focused panel: 2px left border `#6c63ff` on the panel container
- Unfocused panels: no left border (transparent/none)
- Keyboard shortcuts route to the focused panel first, then fall through to global handlers

## Interaction Contract

### Transport Controls

#### Play Button
- Icon: lucide `Play` (16px) when stopped, lucide `Pause` (16px) when playing
- Size: 32x32px, rounded-md
- Stopped state: `#3a3a5a` background, `#eeeeee` icon
- Playing state: `#6c63ff` background, `#ffffff` icon
- Click: Toggle play/stop via `window.calliope.transportPlay()` / `window.calliope.transportStop()`
- Keyboard: Space bar (global, highest priority)

#### Stop Button
- Icon: lucide `Square` (16px)
- Size: 32x32px, rounded-md
- Default: `#3a3a5a` background, `#eeeeee` icon
- Hover: `#4a4a6a` background
- Click: Stop transport and reset position to 0 via `window.calliope.transportStop()`

#### Record Button
- Icon: filled circle (CSS `w-3 h-3 rounded-full`)
- Size: 32x32px, rounded-md
- Inactive: `#3a3a5a` background, `#999999` circle
- Active (recording): `#ef4444` background, `#ffffff` circle, subtle pulse animation (CSS `animate-pulse` at 1s interval)
- Click: Toggle record arm via `window.calliope.transportRecord()`
- Keyboard: R (global, when no input focused)

### BPM Display

- Display: 16px semibold `#eeeeee` for the numeric value, 10px regular `#999999` "BPM" label below
- Interaction: Click on the BPM value to enter edit mode
- Edit mode: Replace text with a 72px-wide `<input type="number">` with min=20 max=300 step=1
- Edit styling: `#1a1a2e` background, `#eeeeee` text, 1px `#6c63ff` border (focus ring)
- Confirm: Enter key or blur commits the value via `window.calliope.commandDispatch('transport.bpm', newValue)`
- Cancel: Escape key reverts to previous value, exits edit mode
- No scroll-to-adjust (user decision: click to type)

### Time Signature Display

- Display: 16px semibold `#eeeeee` for numerator, "/" separator, 16px semibold for denominator
- Read-only in this phase (editable via command dispatcher directly)
- Values sourced from `useTimelineStore` transport state

### Position Display

- Two formats side by side, separated by a 1px `#3a3a5a` vertical divider:
  - **Time**: MM:SS.ms in 16px semibold `#eeeeee`, updates at ~60fps from rAF playhead sync
  - **Bars:Beats**: "001:01" format in 16px semibold `#eeeeee`, computed from beat position + time signature
- Label: 10px `#999999` "TIME" and "POSITION" below each readout
- Monospace font for numeric displays: `font-variant-numeric: tabular-nums` (prevents jitter during playback)

### Loop Toggle

- Icon: lucide `Repeat` (16px)
- Size: 32x32px, rounded-md
- Inactive: `#3a3a5a` background, `#999999` icon
- Active: `#6c63ff` background, `#ffffff` icon
- Click: Toggle loop mode via `window.calliope.transportSetLoop()`
- Keyboard: L (global, when no input focused)

### Metronome Toggle

- Icon: lucide `Timer` (16px)
- Size: 32x32px, rounded-md
- Inactive: `#3a3a5a` background, `#999999` icon
- Active: `#6c63ff` background, `#ffffff` icon
- Click: Toggle metronome via `window.calliope.metronomeToggle()`
- Keyboard: None (not frequent enough for shortcut)

### Panel Focus System

- Implementation: Zustand store field `focusedPanel: 'timeline' | 'piano-roll' | 'mixer'`
- Default: `'timeline'`
- Trigger: `pointerdown` event on any panel container sets focus
- Visual: Focused panel container gains `border-l-2 border-[#6c63ff]`; unfocused panels have `border-l-2 border-transparent`
- Keyboard routing: Global keydown handler checks `focusedPanel` before dispatching to panel-specific handlers

### Waveform Rendering (Audio Clips)

- Already implemented in `AudioClip.tsx` with PixiJS Graphics `poly()` for peak data
- Enhancement: Ensure `clip.peakData` is populated from C++ engine when audio files are loaded
- Peak data format: Float32Array of normalized values (-1.0 to 1.0), one value per pixel at current zoom
- Rendering: Closed polygon approach (top peaks + mirrored bottom), track color at 50% alpha
- No-data fallback: Dashed center line (already implemented)
- Performance: `cullable={true}` on clip containers (already set), PixiJS handles WebGL batching

### Unified Layout

- Layout order (top to bottom):
  1. `TransportBar` -- 48px, fixed, always visible
  2. `TimelineView` -- flex-1, min-height 200px
  3. `SplitDivider` + `PianoRollPanel` -- conditional, 200px min, 60vh max
  4. `SplitDivider` + `MixerPanel` -- conditional, 240px min, 60vh max
- Engine status indicator: absolute bottom-right, 10px `#666666` (existing, no change)
- Window title: "{projectName} - LuneyTunes" via Electron `document.title` update from project-store

## Keyboard Shortcuts (Consolidated)

### Global (always active, highest priority)

| Shortcut | Action |
|----------|--------|
| Space | Play / Stop toggle |
| R | Record toggle |
| L | Loop toggle |
| Ctrl+Z | Undo |
| Ctrl+Shift+Z | Redo |
| Ctrl+S | Save project |
| Ctrl+Shift+E | Open export dialog |

### Timeline Panel (when focused)

| Shortcut | Action |
|----------|--------|
| Delete / Backspace | Delete selected clips |
| Ctrl+A | Select all clips |
| Ctrl+D | Duplicate selected clips |
| Escape | Clear selection |
| S | Toggle snap |
| Ctrl+= | Zoom in |
| Ctrl+- | Zoom out |

### Piano Roll Panel (when focused, capture-phase priority)

| Shortcut | Action |
|----------|--------|
| Delete / Backspace | Delete selected notes |
| Ctrl+A | Select all notes |
| Ctrl+D | Duplicate selected notes |
| Escape | Clear selection / close piano roll |
| Q | Quantize selected notes |

### Mixer Panel (when focused)

| Shortcut | Action |
|----------|--------|
| M | Toggle mute on selected track |
| S | Toggle solo on selected track |

Note: On macOS, Ctrl maps to Cmd (metaKey). Shortcuts are suppressed when focus is in an `<input>`, `<textarea>`, or `contentEditable` element.
