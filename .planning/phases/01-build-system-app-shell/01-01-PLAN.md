       src/bridge.cpp

   set_target_properties(calliope_addon PROPERTIES PREFIX "" SUFFIX ".node")
   target_link_libraries(calliope_addon PRIVATE calliope_engine ${CMAKE_JS_LIB})
   execute_process(
       COMMAND node -p "require('node-addon-api').include"
       WORKING_DIRECTORY ${CMAKE_SOURCE_DIR}
       OUTPUT_VARIABLE NODE_ADDON_API_DIR
       OUTPUT_STRIP_TRAILING_WHITESPACE

   target_include_directories(calliope_addon PRIVATE ${NODE_ADDON_API_DIR})

  <action>
1. Create `engine/include/calliope/engine.h` with namespace calliope, class Engine:
   - `static std::string getJuceVersion();`
   - `static std::vector&lt;std::string&gt; getAudioDevices();`
   - `static bool startTestTone(double frequency, int deviceIndex = -1);`
   - `static void stopTestTone();`
   - Private constructor (static-only class)
   - Include: string, vector, memory

2. Create `engine/src/engine.cpp`:
   - `getJuceVersion()`: returns `juce::SystemStats::getJUCEVersion().toStdString()`
   - `getAudioDevices()`: calls `juce::MessageManager::getInstance()` first (per Pitfall 3), creates a local `juce::AudioDeviceManager`, iterates `getAvailableDeviceTypes()`, calls `scanForDevices()`, collects device names as "TypeName: DeviceName" strings
   - Include juce_core/juce_core.h and juce_audio_devices/juce_audio_devices.h

3. Create `engine/src/test_tone.cpp`:
   - Use a static `std::unique_ptr&lt;juce::AudioDeviceManager&gt;` and a simple AudioSourcePlayer + ToneGeneratorAudioSource, OR implement a minimal juce::AudioIODeviceCallback subclass that writes a sine wave into the output buffer.
   - `startTestTone(frequency, deviceIndex)`:
     - Ensure MessageManager is initialized
     - Create AudioDeviceManager if not exists
     - Initialize with default output device (2 output channels, 0 input)
     - Create a simple callback that generates `sin(2 * pi * frequency * sampleIndex / sampleRate)` into each output channel
     - Add the callback to the device manager
     - Return true on success, false on failure
   - `stopTestTone()`:
     - Remove audio callback
     - Close the audio device
     - Reset the AudioDeviceManager
   - Use `juce::dsp::Oscillator` if simpler, or manual sine calculation (both are acceptable, but manual is more self-contained for Phase 1)

4. Create `native/src/bridge.h`:
   - Forward declare: `Napi::Value GetEngineInfo(const Napi::CallbackInfo& info);`
   - Forward declare: `Napi::Value StartTestTone(const Napi::CallbackInfo& info);`
   - Forward declare: `Napi::Value StopTestTone(const Napi::CallbackInfo& info);`

5. Create `native/src/bridge.cpp` (per D-08, Research Pattern 2):
   - `GetEngineInfo`: creates Napi::Promise::Deferred, creates ThreadSafeFunction, spawns std::thread that calls calliope::Engine::getJuceVersion() and calliope::Engine::getAudioDevices(), uses tsfn.BlockingCall to resolve promise with {juceVersion: string, audioDevices: string[]}. Releases TSFN after.
   - `StartTestTone`: takes frequency argument (info[0].As&lt;Napi::Number&gt;().DoubleValue()), creates promise, spawns thread that calls calliope::Engine::startTestTone(frequency), resolves with success boolean.
   - `StopTestTone`: creates promise, spawns thread that calls calliope::Engine::stopTestTone(), resolves with undefined.
   - Include napi.h, bridge.h, calliope/engine.h, thread

6. Create `native/src/addon.cpp`:
   - Include napi.h and bridge.h
   - `Napi::Object Init(Napi::Env env, Napi::Object exports)` that sets:
     - exports.Set("getEngineInfo", Napi::Function::New(env, GetEngineInfo))
     - exports.Set("startTestTone", Napi::Function::New(env, StartTestTone))
     - exports.Set("stopTestTone", Napi::Function::New(env, StopTestTone))
   - `NODE_API_MODULE(calliope_addon, Init)`

7. Build the native addon: `pnpm run build:native`
   - If cmake-js fails to find Electron headers, ensure the cmake-js config in package.json is correct and try `npx cmake-js build --runtime=electron --runtime-version=35.7.5`
   - If JUCE link errors about GUI/Cocoa, verify only the specified modules are linked (no juce_gui_basics)
   - If MessageManager assertions fire, ensure getInstance() is called before AudioDeviceManager operations

8. Verify the addon loads: `node -e "const addon = require('./build/Release/calliope_addon.node'); addon.getEngineInfo().then(info => console.log(JSON.stringify(info)))"`
   - Note: This test runs in plain Node.js, not Electron. The addon is built against Electron headers so this may fail with ABI mismatch. That is expected -- the real test is loading in Electron (Plan 02). If it works in Node.js too, great.
