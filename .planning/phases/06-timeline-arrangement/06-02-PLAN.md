    1. **TimelineCanvas.tsx** — PixiJS Application with pixi-viewport (per D-01, D-04, research Pattern 1):
       - Import `'../timeline/pixi-setup'` at top of file (triggers extend() registration)
       - Import `{ Application, useApplication }` from `'@pixi/react'`
       - Import `{ Viewport }` from `'pixi-viewport'` and call `extend({ Viewport })` at module scope
       - Declare pixi-viewport TypeScript types:
         ```typescript
         declare module '@pixi/react' {
           interface PixiElements {
             viewport: import('@pixi/react').PixiReactElementProps<typeof Viewport>
           }
         }
         ```
       - Accept a `containerRef: React.RefObject<HTMLDivElement>` prop for sizing
       - Use `<Application resizeTo={containerRef.current}>` with `background="#1a1a2e"` (dominant color)
       - Inside Application, render a `<viewport>` component with:
         - screenWidth/screenHeight from container dimensions (use ResizeObserver or state)
         - worldWidth: 10000 (large enough for long arrangement, can be dynamic later)
         - worldHeight: tracks.length * 80 (80px per track row)
         - events: `application.renderer.events` (required for pixi-viewport v6)
         - Configure viewport plugins: `.drag({ mouseButtons: 'middle' })`, `.wheel({ smooth: 5 })`, `.clamp({ direction: 'all' })`, `.decelerate()`
         - Ctrl/Cmd + wheel: horizontal zoom (per D-04) — handle in a viewport `wheel` event listener, adjust `setPixelsPerBeat`
         - Plain wheel: vertical scroll — update `setScrollY`
         - Shift + wheel: horizontal scroll — update `setScrollX`
         - On viewport `moved` event: sync scrollX/scrollY back to Zustand store
       - Render `<GridLayer />` inside viewport
       - NOTE: Clip components (MidiClip, AudioClip) will be added in Plan 03

    2. **GridLayer.tsx** — PixiJS component for grid lines (per D-12, TL-05):
       - Renders vertical grid lines inside the viewport
       - Uses `<pixiGraphics>` with a draw callback
       - Reads `pixelsPerBeat`, `gridResolution`, `timeSignature` from store
       - Draws major lines (bar boundaries) at `rgba(255,255,255,0.12)` (per UI-SPEC grid_line_major), 1px wide
       - Draws minor lines (beat subdivisions matching gridResolution) at `rgba(255,255,255,0.05)` (per UI-SPEC grid_line_minor), 1px wide
       - Only draw visible grid lines based on viewport bounds (performance optimization)
       - Horizontal track divider lines at every 80px (track row boundaries)
       - Use Graphics draw callback pattern: `<pixiGraphics draw={(g) => { g.clear(); /* draw lines */ }} />`

    3. **Playhead.tsx** — CSS-positioned div overlay (per D-03):
       - Absolute positioned div, 1px wide, `bg-[#6c63ff]` (accent color), full height of canvas area
       - Position: `left` set to `beatToPixel(currentBeat, pixelsPerBeat, scrollX) + 200` (200px offset for track header column)
       - Uses the `usePlayhead` hook for position
       - Only visible when position is within the visible area
       - z-index above canvas but below context menus

    4. **use-playhead.ts** — requestAnimationFrame hook for engine transport sync (per D-03, research Pattern 5):
       - Runs a rAF loop that calls `window.calliope.getTransportState()`
       - Reads ppqPosition from transport state
       - Updates store: `setTransportState({ isPlaying: state === 'playing', currentBeat: ppqPosition, bpm, timeSignature })`
       - Cleanup: cancel rAF on unmount
       - Throttle: only call getTransportState every 2nd frame (30Hz) to reduce IPC overhead
       - Handle case where window.calliope is not available (graceful fallback for development without engine)

    5. **Update TimelineView.tsx** — Wire in TimelineCanvas and Playhead:
       - Add a `ref` for the canvas container div
       - Replace placeholder with `<TimelineCanvas containerRef={canvasContainerRef} />`
       - Add `<Playhead />` as absolute-positioned overlay
       - Call `usePlayhead()` hook in TimelineView to start rAF sync

    IMPORTANT per research Pitfall 6: Configure pixi-viewport wheel events carefully. Use `passiveWheel: false` and scope event handling to the canvas element. Prevent Ctrl+wheel from being consumed by viewport's default zoom — handle it manually to control `pixelsPerBeat`.
