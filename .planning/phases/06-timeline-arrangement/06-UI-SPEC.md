# Phase 6 — UI Design Contract

> Visual and interaction contract for the Timeline & Arrangement view. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom components — DAW UI has no standard library match) |
| Icon library | lucide-react (lightweight, tree-shakable, MIT) |
| Font | system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif |

**Rationale:** shadcn/radix component libraries are not initialized and would add overhead for a phase that is 90% canvas (PixiJS). The non-canvas chrome (track headers, toolbar, context menus) is simple enough for custom Tailwind components. Material UI and Ant Design explicitly rejected in CLAUDE.md.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, grid line labels, inline padding |
| sm | 8px | Track header internal padding, button gaps, clip label padding |
| md | 16px | Toolbar padding, context menu item padding, section gaps |
| lg | 24px | Track header height padding, panel margins |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Timeline ruler height (48px) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Track header width: 200px (fixed, per D-02)
- Track row height: 80px default (divisible by 4, fits waveform display and clip labels)
- Timeline ruler height: 48px (houses beat numbers, bar markers, loop region handle)
- Minimum clip width: 8px (1/32 note at default zoom, prevents invisible clips)
- Touch/click target minimum: 44px height for toolbar buttons (accessibility)

## Component Inventory

### Canvas Layer (PixiJS)

These render inside a single PixiJS `<Stage>` component via `@pixi/react`:

| Component | Responsibility |
|-----------|---------------|
| `TimelineCanvas` | Root PixiJS stage — manages viewport, scroll, zoom, grid rendering |
| `GridLayer` | Draws vertical grid lines (bar/beat/subdivision) scaled to zoom level |
| `ClipContainer` | Container per track row — positions clips horizontally by beat |
| `MidiClip` | Rounded rect with top color bar, label text, mini note preview lines |
| `AudioClip` | Rounded rect with top color bar, label text, waveform peaks rendering |
| `SelectionBox` | Rubber-band rectangle drawn during drag-select |
| `LoopRegion` | Shaded overlay on timeline ruler showing loop start/end with drag handles |

### DOM Layer (React + Tailwind)

| Component | Responsibility |
|-----------|---------------|
| `TimelineView` | Top-level layout — track headers (left) + canvas (right) + ruler (top) |
| `TimelineRuler` | Horizontal bar/beat number ruler above canvas (DOM, not canvas) |
| `TrackHeaderList` | Scrollable column of track headers, syncs vertical scroll with canvas |
| `TrackHeader` | Track name (editable), color strip, mute/solo/arm buttons |
| `TimelineToolbar` | Grid resolution dropdown, snap toggle, zoom controls |
| `Playhead` | Absolute-positioned `div`, 1px wide, `#6c63ff`, full timeline height, updated via `requestAnimationFrame` |
| `TrackContextMenu` | Right-click menu: Add Track, Delete Track, Rename, Change Color, Duplicate |
| `ClipContextMenu` | Right-click menu: Delete Clip, Duplicate, Split, Change Color |
| `GridResolutionSelect` | Dropdown: 1/4, 1/8, 1/16, 1/32 note values |
| `AddTrackButton` | "+" button below last track header |

### Layout Structure

```
+----------------------------------------------------------+
|  TimelineToolbar (48px height)                           |
|  [Grid: 1/4 v] [Snap: On] [Zoom: -  +]                |
+----------+-----------------------------------------------+
|          |  TimelineRuler (48px height)                   |
|  Track   |  |1    |2    |3    |4    |5    |6    |7       |
|  Headers +-----------------------------------------------+
|  (200px) |                                               |
|  Track 1 |  [===MIDI Clip===]    [==Audio Clip==]        |
|  Track 2 |       [====MIDI Clip====]                     |
|  Track 3 |  [=Audio=]                                    |

|  [+ Add] |                                               |
+----------+-----------------------------------------------+
```

- Track headers scroll vertically in sync with canvas
- Timeline ruler scrolls horizontally in sync with canvas
- Canvas occupies remaining space (flex-grow)
- Playhead overlays the canvas + ruler as an absolute-positioned div

## Interaction Contracts

### Clip Creation (D-05)
1. User clicks and drags on empty track area
2. Ghost clip appears at drag start, extends to current mouse position
3. Ghost clip snaps to grid resolution on both edges
4. On mouse release: dispatch `clip:create` command with `{trackId, startBeat, lengthBeats, type: "midi"}`
5. Minimum drag distance: 1 grid unit (prevents accidental creation)

### Clip Selection (D-06)
1. Click clip: select it (deselect all others)
2. Shift+click clip: add to selection
3. Ctrl/Cmd+click clip: toggle in selection
4. Click empty area: deselect all
5. Drag on empty area: rubber-band selection (selects all clips intersecting the rectangle)
6. Selected clips show `#6c63ff` border (2px) and slight brightness increase

### Clip Move
1. Click and drag selected clip(s)
2. Clips follow mouse, snapped to grid
3. Visual ghost shows destination position
4. Drop dispatches `clip:move` command with new `{startBeat, trackId}` for each clip
5. Escape during drag: cancel move

### Clip Resize (D-08)
1. Mouse within 6px of clip left/right edge: cursor changes to `col-resize`
2. Drag edge: clip resizes, snapped to grid
3. Minimum clip length: 1 grid unit
4. Dispatch `clip:resize` command with new `{startBeat, lengthBeats}`

### Horizontal Zoom (D-04)
1. Ctrl/Cmd + scroll wheel: zoom in/out horizontally
2. Zoom center: mouse cursor position
3. Zoom range: 8 beats visible (max zoom) to 256 beats visible (min zoom)
4. Grid lines appear/disappear based on zoom level (subdivisions only visible when space permits)
5. Zoom level stored in Zustand, affects `pixelsPerBeat` calculation

### Scroll
1. Scroll wheel: vertical scroll (track rows)
2. Shift + scroll wheel: horizontal scroll (timeline)
3. Scrollbar sync: track header list and canvas share vertical scroll position
4. Timeline ruler and canvas share horizontal scroll position

### Track Management (D-07)
1. Right-click track header: context menu appears at cursor
2. Context menu items: Add Track Above, Add Track Below, Delete Track, Rename Track, Change Color (submenu with 8 swatches), Duplicate Track
3. "+" button at bottom of track list: adds new track at end
4. Double-click track name: inline edit mode (input field replaces label, Enter to confirm, Escape to cancel)
5. Drag track header: reorder (vertical drag with insertion indicator line)

### Playhead (D-03)
1. Thin vertical line (`1px`, `#6c63ff`) spanning from ruler to bottom of last track
2. Position updated via `requestAnimationFrame` reading engine transport position
3. Click on ruler: move playhead to clicked beat position (snapped to grid)
4. During playback: playhead moves smoothly, auto-scrolls canvas when reaching 80% of visible width

### Loop Region (TL-06)
1. Shift+click+drag on ruler: create loop region
2. Loop region shown as tinted overlay (`#6c63ff` at 15% opacity) on ruler and faintly on canvas
3. Drag loop edges to resize
4. Double-click loop region to remove it
5. Dispatch `transport:setLoopRegion` with `{startBeat, endBeat}`

## State Shape (Zustand)

Reference for executor — the Zustand store shape this phase needs:

```typescript
interface TimelineState {
  // Tracks
  tracks: Track[]

  // Clips
  clips: Record<string, Clip>  // clipId -> Clip

  // Viewport
  scrollX: number        // horizontal scroll in pixels
  scrollY: number        // vertical scroll in pixels
  pixelsPerBeat: number  // zoom level (default: 24)

  // Grid
  gridResolution: 0.25 | 0.125 | 0.0625 | 0.03125  // in beats (1/4, 1/8, 1/16, 1/32)
  snapEnabled: boolean

  // Selection
  selectedClipIds: Set<string>
  selectedTrackId: string | null

  // Loop
  loopRegion: { startBeat: number; endBeat: number } | null

  // Transport (mirrored from engine)
  isPlaying: boolean
  currentBeat: number
  bpm: number
  timeSignature: { numerator: number; denominator: number }
}

interface Track {
  id: string
  name: string
  colorIndex: number  // 0-7, indexes into track color palette
  muted: boolean
  solo: boolean
  armed: boolean
  order: number
}

interface Clip {
  id: string
  trackId: string
  type: 'midi' | 'audio'
  startBeat: number
  lengthBeats: number
  name: string
  // Audio clips only
  peakData?: Float32Array  // pre-computed waveform peaks from engine
}
```

## Performance Targets

| Metric | Target | Method |
|--------|--------|--------|
| Canvas frame rate | 60fps during scroll/zoom | PixiJS WebGL, minimize draw calls |
| Playhead update | 60fps smooth motion | requestAnimationFrame, CSS transform |
| Clip render count | 200+ clips without jank | PixiJS viewport culling (only render visible clips) |
| Waveform peaks | Pre-computed by C++ engine | Avoid decoding audio in renderer |
| Track header scroll sync | No visible desync | Shared scroll position in Zustand |
