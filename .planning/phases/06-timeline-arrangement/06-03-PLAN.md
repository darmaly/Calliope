    1. **MidiClip.tsx** — PixiJS component for MIDI clip rendering (per UI-SPEC MidiClip spec):
       - Props: `clip: Clip`, `trackColorIndex: number`, `pixelsPerBeat: number`, `isSelected: boolean`, `trackRowY: number`
       - Uses `<pixiGraphics>` with draw callback
       - Position: x = `clip.startBeat * pixelsPerBeat`, y = `trackRowY`
       - Width: `clip.lengthBeats * pixelsPerBeat`, height: 72px (80px track - 8px padding)
       - Draw a rounded rectangle (radius 4px):
         - Fill: `TRACK_COLORS[trackColorIndex]` at 30% opacity (0x4a9eff -> use hex with alpha) for body
         - Top bar: 4px tall strip at full color opacity
         - Border: if selected, 2px `#6c63ff` (accent) border (per UI-SPEC selection style)
       - Clip name text: rendered via `<pixiText>` at 11px, `#eeeeee`, positioned inside clip with 4px padding
       - Set `eventMode = 'static'` and `cursor = 'pointer'` for interactivity (needed for Plan 04 click handlers)
       - Minimum display width: 8px (per UI-SPEC spacing)

    2. **AudioClip.tsx** — PixiJS component for audio clip with waveform (per TL-03, UI-SPEC AudioClip spec):
       - Same structure as MidiClip but with waveform rendering
       - Props: same as MidiClip
       - Draw rounded rectangle same as MidiClip but fill color uses `#22c55e` (audio clip color from UI-SPEC) at 30% opacity
       - If `clip.peakData` exists: call `peaksToVertices(clip.peakData, clipWidth, clipHeight - 8)` and render as a `<pixiGraphics>` polygon filled with the track color at 50% opacity
       - If no peakData: render an empty clip body with dashed center line
       - Top bar: 4px tall at full `#22c55e`
       - Selected border: same as MidiClip

    3. **ClipContainer.tsx** — PixiJS container for a single track row's clips:
       - Props: `track: Track`, `trackIndex: number`
       - Reads clips from store filtered by `clip.trackId === track.id`
       - Use Zustand selector: `useTimelineStore(s => Object.values(s.clips).filter(c => c.trackId === track.id))`
       - Read `pixelsPerBeat` and `selectedClipIds` from store
       - Position: `<pixiContainer y={trackIndex * 80}>` (80px per track row)
       - Renders `<MidiClip>` or `<AudioClip>` based on clip.type
       - Set `cullable={true}` on each clip for performance (per research Don't Hand-Roll: viewport culling)

    4. **SelectionBox.tsx** — Rubber-band selection rectangle (per D-06):
       - Props: `startX: number`, `startY: number`, `endX: number`, `endY: number`, `visible: boolean`
       - Uses `<pixiGraphics>` draw callback
       - Dashed rectangle outline: `#6c63ff` (accent) at 100% opacity, 1px stroke
       - Fill: `#6c63ff` at 10% opacity
       - Only renders when `visible` is true

    5. **LoopRegion.tsx** — Loop overlay on timeline (per TL-06, UI-SPEC interaction contract):
       - Reads `loopRegion` and `pixelsPerBeat` from store
       - If loopRegion is null, render nothing
       - Uses `<pixiGraphics>` draw callback
       - Draws a filled rectangle from `loopRegion.startBeat * pixelsPerBeat` to `loopRegion.endBeat * pixelsPerBeat`
       - Fill: `#6c63ff` at 15% opacity (per UI-SPEC)
       - Height: full world height (all tracks)
       - Bracket lines at start and end: 2px wide, `#6c63ff` at 80% opacity

    NOTE: These components are self-contained PixiJS elements. They will be integrated into TimelineCanvas in Plan 04 (Task 1 wires them into the viewport alongside interaction handlers).
