| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, knob label inset, fader tick marks |
| sm | 8px | Compact element spacing, button gaps within strip |
| md | 16px | Default element spacing, padding inside channel strips |
| lg | 24px | Section padding, gap between effect slots |
| xl | 32px | Layout gaps |
| 2xl | 48px | Toolbar height (matches Phase 6/7 toolbar) |
| 3xl | 64px | Channel strip width (track strips) |

Exceptions:
- Track channel strip width: 64px (compact vertical strip)
- Master channel strip width: 88px (wider, visually distinct)
- Fader track height: 160px (vertical slider travel)
- Level meter width: 8px per channel (16px total for stereo L+R)
- Pan knob diameter: 32px
- Touch target minimum: 24px for M/S/Arm buttons (reuse Phase 6 pattern)

### Accent Reserved For

- Active solo button glow (solo is `#22c55e`, accent not used here)
- Selected/focused channel strip border (2px left border `#6c63ff`)
- Mixer toggle button active state in timeline toolbar
- Effect parameter popover focus ring

Note: Mute/Solo/Arm buttons reuse Phase 6 TrackHeader colors exactly:
- Mute active: `#f59e0b` (amber)
- Solo active: `#22c55e` (green)
- Arm active: `#ef4444` (red)
- Inactive: `#3a3a5a` background, `#999999` text

### Level Meter Colors

Canvas 2D gradient from bottom to top:
- `#22c55e` (green): -inf to -12dB (safe zone, bottom 60%)
- `#f59e0b` (amber): -12dB to -3dB (caution zone, next 30%)
- `#ef4444` (red): -3dB to +6dB (danger zone, top 10%)

Peak hold indicator: `#eeeeee` (1px horizontal line, decays after 1.5s)
Meter background: `#1a1a2e` (dominant color)
Meter border: `#3a3a5a` (1px)

### Master Strip Visual Distinction

Master strip background: `#2a2a4e` (slightly lighter than track strips to differentiate)
Master strip left border: 2px solid `#6c63ff` (accent, always visible)
Master strip label: "Master" in 13px medium weight `#eeeeee`

## Visual Hierarchy

**Primary focal point:** Faders and level meters are the primary work area — the user scans these to understand the mix balance at a glance.

**Reading order:**
1. **Toolbar** (top) — mixer toggle, view controls
2. **Channel strips** (left-to-right) — per-track fader/pan/buttons, horizontal scroll for overflow
3. **Master strip** (rightmost, always visible) — final output level and processing

## Interaction Contract

### Volume Fader

- DOM element: Custom vertical range input styled with Tailwind
- Range: -infinity to +6dB (mapped as linear slider 0-1.0 internally, displayed as dB)
- dB scale markings at: +6, 0, -6, -12, -24, -48, -inf
- Default position: 0dB (unity gain)
- Interaction: Pointer drag on thumb, click on track to jump
- Double-click thumb: Reset to 0dB
- Visual: 4px wide track, 12px wide x 20px tall thumb, `#eeeeee` thumb color
- Live readout: Current dB value displayed below fader in 9px text
- Dispatches: `command:dispatch` with `track.{trackId}.volume` parameter

### Pan Knob

- DOM element: CSS-rotated circular indicator with pointer drag
- Range: -100L (fully left) to +100R (fully right), center = 0
- Rotation: 270-degree arc (-135deg to +135deg)
- Interaction: Pointer drag vertically (up = right, down = left)
- Double-click: Reset to center (0)
- Display: Current value as text below knob ("C", "50L", "100R")
- Knob visual: 32px circle, `#3a3a5a` background, 2px `#eeeeee` indicator line from center to edge
- Dispatches: `command:dispatch` with `track.{trackId}.pan` parameter

### Mute / Solo / Arm Buttons

- Reuse exact Phase 6 TrackHeader button pattern
- 24x24px buttons, same color scheme (amber/green/red)
- Layout: Horizontal row within channel strip
- Same store actions: `toggleMute`, `toggleSolo`, `toggleArm`

### Effect Insert Slots

- Each channel strip shows up to 8 effect slots vertically
- Empty slot: Click to show dropdown with available effects (EQ, Compressor, Reverb, Delay, Limiter)
- Filled slot: Shows effect name, bypass toggle (dot icon), click to open param popover
- Bypass toggle: Filled circle = active, empty circle = bypassed (reuse Arm button pattern with `#22c55e` active)
- Reorder: Drag effect slot vertically to reorder within the chain
- Remove: Right-click context menu "Remove Effect"
- Dispatches: `effectInsert`, `effectRemove`, `effectReorder`, `effectBypass` via existing IPC bridge

### Effect Parameter Popover

- Trigger: Click on filled effect slot
- Rendering: Portal-based popover positioned above/below the slot
- Content: List of parameter sliders for the effect (horizontal range inputs)
- Each slider: Parameter name (10px label), horizontal slider, current value
- Close: Click outside (document click listener, same as ContextMenu pattern)
- Dispatches: `command:dispatch` with `effects.{trackId}.{slotIndex}.{paramName}` parameters

### Level Meter

- Canvas 2D element, 16px wide (8px L + 8px R), full fader height
- Update rate: ~30Hz via requestAnimationFrame, data polled from C++ engine via IPC
- Smoothing: Ballistic meter with 300ms attack, 500ms release (exponential decay)
- Peak hold: White 1px line at peak level, holds for 1.5s then drops
- Clip indicator: If peak reaches 0dB, top 2px turns `#ef4444` and stays until clicked
- Click on meter: Reset clip indicator

### Channel Strip Selection

- Click on strip header (name area): Select track in timeline store
- Visual: 2px left border `#6c63ff` on selected strip
- Sync: `selectedTrackId` in timeline store

### Horizontal Scrolling

- Mouse wheel on strip area: Scroll horizontally through channel strips
- Master strip is always visible (outside scroll container, pinned right)
- Scrollbar: Native horizontal scrollbar, styled with Tailwind `scrollbar-thin`

### Mixer Panel Toggle

- Toolbar button in TimelineToolbar: Lucide `SlidersHorizontal` icon + "Mixer" text
- Pattern: Same as existing Piano Roll toggle button
- Active state: `text-[#6c63ff] underline` (matches Piano Roll toggle)

## State Contract

### Timeline Store Extensions

Track type gains two new fields:

```typescript
export interface Track {
  // ... existing fields
  volume: number   // 0.0 to ~2.0 (0dB = 1.0, +6dB ~= 2.0), default 1.0
  pan: number      // -1.0 (full left) to +1.0 (full right), default 0.0
}
```

### New Zustand Store: `mixer-store.ts`

| State Field | Type | Default | Purpose |
|-------------|------|---------|---------|
| `panelVisible` | `boolean` | `false` | Mixer panel open/closed |
| `panelHeight` | `number` | `300` | Mixer panel height in pixels |
| `meterLevels` | `Record<string, { left: number; right: number }>` | `{}` | Per-track meter RMS levels (0.0-1.0) |
| `meterPeaks` | `Record<string, { left: number; right: number }>` | `{}` | Per-track peak hold levels |
| `masterMeterLevel` | `{ left: number; right: number }` | `{ left: 0, right: 0 }` | Master meter RMS |
| `masterMeterPeak` | `{ left: number; right: number }` | `{ left: 0, right: 0 }` | Master meter peak |
| `effectChains` | `Record<string, EffectSlot[]>` | `{}` | Per-track effect insert chains |
| `masterEffectChain` | `EffectSlot[]` | `[]` | Master bus effect chain |

### EffectSlot Type

```typescript
interface EffectSlot {
  id: string          // crypto.randomUUID()
  effectType: 'eq' | 'compressor' | 'reverb' | 'delay' | 'limiter'
  bypassed: boolean
  label: string       // Display name: "Parametric EQ", "Compressor", etc.
}
```
