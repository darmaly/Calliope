    Tests need a Transport instance but NOT a full Engine (avoid audio device init in unit tests). For transport commands that call Engine::getInstance().getTransport(), the tests should initialize the Engine first OR the commands should accept a Transport reference.

    **Design decision:** Commands accept references to the component they operate on (Transport&, MetronomeProcessor&, MasterBusProcessor&) in their constructor rather than calling Engine::getInstance() directly. This makes them testable without audio hardware. The Engine wrappers (Plan 02 Task 2) will pass the correct references when constructing commands.

    For transport command tests, create a Transport instance directly (it has no dependencies on audio hardware). For metronome/master bus tests, create MetronomeProcessor/MasterBusProcessor instances directly (their atomic members are accessible without prepareToPlay).

    **Step 4: Create transport_commands.h/.cpp**

    Define in `engine/include/calliope/commands/transport_commands.h`:

    Non-undoable commands (override `isUndoable()` to return false):
    - `PlayCommand(Transport& transport)` — perform() calls transport.play()
    - `StopCommand(Transport& transport)` — perform() calls transport.stop()
    - `PauseCommand(Transport& transport)` — perform() calls transport.pause()

    Undoable commands (default isUndoable() = true):
    - `SetBpmCommand(Transport& transport, double newBpm)` — perform() captures oldBpm_ from transport.getBpm() then calls transport.setBpm(newBpm_). undo() calls transport.setBpm(oldBpm_). getEventData() returns {"bpm": newBpm_}. getCommandName() returns "SetBpm".
    - `SetTimeSignatureCommand(Transport& transport, int newNum, int newDen)` — perform() captures old values, sets new. undo() restores. getEventData() returns {"numerator": N, "denominator": D}.
    - `SetLoopRegionCommand(Transport& transport, double startBeat, double endBeat, bool enabled)` — perform() captures old loop settings (loopStartBeat, loopEndBeat, isLooping), sets new. undo() restores all three.
    - `SetMetronomeEnabledCommand(MetronomeProcessor& metronome, bool enabled)` — perform() captures old enabled.load(), sets new. undo() restores.
    - `SetMetronomeVolumeCommand(MetronomeProcessor& metronome, float volume)` — perform() captures old volume.load(), sets new. undo() restores.
    - `SetMasterVolumeCommand(MasterBusProcessor& masterBus, float volume)` — perform() captures old masterVolume.load(), sets new. undo() restores.

    All undo() methods return true. All perform() methods return true. All getSizeInUnits() return 1.

    **Step 5: Create parameter_commands.h/.cpp**

    Define `SetParameterCommand(ParameterRegistry& registry, const juce::String& paramId, juce::var newValue)`:
    - perform(): Look up paramId in registry. If not found, return false. Call getter to capture oldValue_. Call setter with newValue_. Return true.
    - undo(): Call setter with oldValue_. Return true.
    - getCommandName(): Return "SetParameter:" + paramId_
    - getEventData(): Return {"parameterId": paramId_, "value": newValue_}
    - isUndoable(): Return true.

    **Step 6: Run tests GREEN**

    // Transport parameters
    paramRegistry_.registerParameter("transport.bpm", {
        [this]() -> juce::var { return getTransport().getBpm(); },
        [this](const juce::var& v) { getTransport().setBpm(static_cast<double>(v)); },
        "double", 20.0, 999.0
    });

    paramRegistry_.registerParameter("transport.timeSigNumerator", {
        [this]() -> juce::var { return getTransport().getTimeSignatureNumerator(); },
        [this](const juce::var& v) { getTransport().setTimeSignature(static_cast<int>(v), getTransport().getTimeSignatureDenominator()); },
        "int", 1, 32
    });

    paramRegistry_.registerParameter("transport.timeSigDenominator", {
        [this]() -> juce::var { return getTransport().getTimeSignatureDenominator(); },

        "int", 1, 32
    });

    // Metronome parameters
    paramRegistry_.registerParameter("metronome.enabled", {
        [this]() -> juce::var { return getAudioGraph().getMetronome().enabled.load(); },
