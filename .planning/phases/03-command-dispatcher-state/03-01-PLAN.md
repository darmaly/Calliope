    **Step 4: Create command_dispatcher.h / .cpp**

    Header `engine/include/calliope/command_dispatcher.h`:
    ```cpp
    #pragma once
    #include <juce_data_structures/juce_data_structures.h>
    #include "calliope/command.h"
    #include "calliope/event_types.h"
    #include <memory>

    namespace calliope {

    class CommandDispatcher {
    public:
        CommandDispatcher();
        ~CommandDispatcher();

        // Execute a command. If undoable, goes through UndoManager.
        // If not undoable, executed directly.
        // Returns true if command was performed.
        bool dispatch(std::unique_ptr<Command> command);

        bool undo();
        bool redo();
        bool canUndo() const;
        bool canRedo() const;
        juce::String getUndoDescription() const;
        juce::String getRedoDescription() const;

        // Listener for command events
        class Listener {
        public:
            virtual ~Listener() = default;
            virtual void commandExecuted(const CommandEvent& event) = 0;
            virtual void commandUndone(const CommandEvent& event) = 0;
        };

        void addListener(Listener* listener);
        void removeListener(Listener* listener);

    private:
        juce::UndoManager undoManager_;
        juce::ListenerList<Listener> listeners_;

        void notifyExecuted(const Command& cmd);
        void notifyUndone(const Command& cmd);
    };

    } // namespace calliope
    ```

    Implementation `engine/src/command_dispatcher.cpp`:
    - Constructor: `undoManager_(0, 200)` — 0 = no size-in-units limit, 200 = keep at least 200 transactions (satisfies PROJ-03 100+ requirement with margin)
    - `dispatch()`: Check `command->isUndoable()`. If true, call `undoManager_.beginNewTransaction(command->getCommandName())` then `undoManager_.perform(command.release())` (UndoManager takes ownership). If false, call `command->perform()` directly (unique_ptr auto-deletes after). Either way, call `notifyExecuted()`.
    - `undo()`: If `canUndo()`, call `undoManager_.undo()`, then `notifyUndone()`. Return success.
    - `redo()`: If `canRedo()`, call `undoManager_.redo()`, then `notifyExecuted()` (redo is a re-execution). Return success.
    - `canUndo()`/`canRedo()`: Delegate to `undoManager_.canUndo()`/`undoManager_.canRedo()`.
    - `getUndoDescription()`/`getRedoDescription()`: Delegate to `undoManager_.getUndoDescription()`/`undoManager_.getRedoDescription()`.
    - `notifyExecuted()`: Build CommandEvent with type=CommandExecuted, commandName from cmd, data from cmd.getEventData(), isUndo=false. Call `listeners_.call(&Listener::commandExecuted, event)`.
    - `notifyUndone()`: Build CommandEvent with type=CommandUndone, isUndo=true. Call `listeners_.call(&Listener::commandUndone, event)`.

    **CRITICAL ownership note:** `juce::UndoManager::perform()` takes a RAW pointer and takes ownership. You MUST call `command.release()` to hand off from unique_ptr. Do NOT hold a reference after release. For non-undoable commands, capture the command name and event data BEFORE calling `perform()`, since the unique_ptr is still valid at that point.
